# Romelly AI - Design System

> Sistema de diseño visual para Romelly AI, plataforma de análisis inteligente de reuniones.

**Filosofía:** Profesional pero amigable. Tech pero humano. Confiable y moderno.

## 2. Tipografía

### Fuentes

| Tipo | Fuente | Google Fonts | Uso |
|------|--------|--------------|-----|
| **Headings** | Inter | [Inter](https://fonts.google.com/specimen/Inter) | Títulos, navegación, botones |
| **Body** | Inter | [Inter](https://fonts.google.com/specimen/Inter) | Texto general, párrafos |
| **Mono** | JetBrains Mono | [JetBrains Mono](https://fonts.google.com/specimen/JetBrains+Mono) | Código, datos, métricas |

> **Por qué Inter:** Es extremadamente legible en pantalla, tiene excelente soporte de features OpenType, y combina profesionalismo con calidez. Su versatilidad permite usarla tanto en headings como en body manteniendo coherencia visual.

### Escala Tipográfica

```css
/* Base: 16px = 1rem */

--text-xs:   0.75rem;    /* 12px - Captions, badges */
--text-sm:   0.875rem;   /* 14px - Labels, metadata */
--text-base: 1rem;       /* 16px - Body text */
--text-lg:   1.125rem;   /* 18px - Lead paragraphs */
--text-xl:   1.25rem;    /* 20px - H5 */
--text-2xl:  1.5rem;     /* 24px - H4 */
--text-3xl:  1.875rem;   /* 30px - H3 */
--text-4xl:  2.25rem;    /* 36px - H2 */
--text-5xl:  3rem;       /* 48px - H1 */
--text-6xl:  3.75rem;    /* 60px - Display */
```

### Pesos

| Peso | Valor | Uso |
|------|-------|-----|
| Regular | 400 | Body text, párrafos |
| Medium | 500 | Labels, subtítulos, buttons |
| Semibold | 600 | Headings H4-H6, énfasis |
| Bold | 700 | Headings H1-H3, brand |

### Line Heights

| Tipo | Valor | Uso |
|------|-------|-----|
| tight | 1.25 | Headings |
| snug | 1.375 | Subtítulos |
| normal | 1.5 | Body text |
| relaxed | 1.625 | Long-form content |

### Tracking (Letter Spacing)

| Tipo | Valor | Uso |
|------|-------|-----|
| tighter | -0.025em | Large headings (H1-H2) |
| tight | -0.015em | Medium headings (H3-H4) |
| normal | 0 | Body text |
| wide | 0.025em | All caps, labels pequeños |

## 3. Logo Conceptual

### Concepto 1: "R Conversacional" ⭐ Recomendado

**Descripción:** 
Una "R" estilizada donde el trazo vertical se transforma en una onda de audio/voz al llegar al centro. La curva de la R fluye naturalmente hacia dos burbujas de conversación sutiles.

**Simbolismo:**
- La "R" representa Romelly (personal, humano)
- La onda de audio = análisis de voz
- Las burbujas = reuniones, conversación
- Fluidez = AI que entiende el contexto

**Colores:**
- Gradient sutil de primary-500 a primary-600
- Versión monocroma en primary-700

**Variantes:**
- Logomark: Solo la R estilizada (favicon, app icon)
- Logotype: R + "romelly" en Inter Semibold
- Full: Logomark + "Romelly AI" con badge "AI" en accent

---

### Concepto 2: "Insight Bubble"

**Descripción:**
Un speech bubble minimalista con forma de diamante o gema en su interior, representando los insights valiosos extraídos de las conversaciones.

**Simbolismo:**
- Bubble = comunicación, reuniones
- Gema interior = insights valiosos, data mining
- Forma geométrica = precisión AI
- Curvas suaves = amigabilidad

**Colores:**
- Bubble: primary-500
- Gema: gradient secondary-400 a accent-300

---

### Concepto 3: "Connected Dots"

**Descripción:**
Tres puntos (representando participantes de reunión) conectados por líneas suaves que forman sutilmente la letra "R". Los puntos tienen diferentes tamaños sugiriendo jerarquía o participación.

**Simbolismo:**
- Puntos = personas en reunión
- Conexiones = comunicación, insights compartidos
- Forma de R = inicial de marca
- Dinamismo = conversación viva

**Colores:**
- Punto principal: primary-500
- Puntos secundarios: secondary-400
- Líneas: gradient entre ambos

## 4. Iconografía

### Estilo Recomendado

**Pack Principal:** [Lucide Icons](https://lucide.dev/)
- Estilo: Outlined, 1.5px stroke
- Tamaño base: 24px (puede escalar a 16, 20, 32)
- Corners: Rounded (match border-radius del sistema)

**Por qué Lucide:**
- Open source, MIT license
- Consistencia en todo el set
- Excelente cobertura de iconos business/tech
- Compatible con React (lucide-react)
- Mismo DNA visual que shadcn/ui

### Iconos por Feature

### Reglas de Uso

1. **Consistencia:** Siempre usar el mismo pack (Lucide)
2. **Tamaño:** 16px (inline), 20px (buttons), 24px (nav), 32px (empty states)
3. **Color:** Heredar del texto padre o usar gray-500/primary-500
4. **Spacing:** Mínimo 8px separación del texto
5. **Hover:** Transición a primary-500 cuando sea interactivo

## 8. Accesibilidad

### Contraste Mínimo
- Texto normal: 4.5:1 (WCAG AA)
- Texto grande (>18px bold): 3:1
- UI components: 3:1

### Focus States
Todos los elementos interactivos deben tener:
- Focus ring visible: 3px primary-200/primary-800
- Never `outline: none` without alternative

### Motion
```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```
